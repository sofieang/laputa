Batch Simulations

A Batch simulation is a collection of simulations, run from a (possibly randomized) initial state for a specific number of steps, while statistics is collected. Since Laputa's model is a stochastic one, it is only batch simulations that can give enough information to draw stable conclusions from. In order for this to be possible, Laputa calculates confidence intervals, and also allows the export of statistics in a form readable by, for instance, Excel.

The batch simulator can work in two different modes, depending on whether it is set to randomize the population or not. When varying population, minimum and maximum number of inquirers can be set, and every simulation is started from a completely random society, with values determined by the parameters of the Batch Simulation window. When population is not varied, the batch simulator begins each simulation with the current society, as it was when the batch simulation was started. This society is then varied randomly, as specified in the parameters. Finally, the simulation is started, and statistics collected.

Some of the parameters in the Batch window (Starting Belief, Inquiry Accuracy, Inquiry Trust, Listen Chance, Threshold and Trust) correspond directly to properties of inquirers and links, and are used to determine starting values of these, unless overridden by individual links' or inquirers' simulation parameters. All of these are represented by various forms of distributions. The other parameters accessible are as follows.

Link Chance: When set to vary this, links are randomized for every new simulation. The link chance is the probability that any order pair a, b of inquirers has a link a->b.

Number of Trials: The number of simulations to run. More simulations gives more certain statistics.

Steps per Trial: The number of steps to run each simulation.

Random Seed: Controls the pseudorandom number generator. Starting two batch simulations with the same initial state, parameters, and random seed give identical results.

Epistemic Value Rules: Sets up how e-value is to be calculated.

Statistics to Record: Controls what kind of statistics to collect. This mainly affects what data it is possible to export.

There are also options to save and load settings for the batch simulation windows, and the same settings can be loaded and saved from the Randomize Society window as well, whose layout is very similar to that of the batch window.
