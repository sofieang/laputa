The Statistics window

The statistics window gives an overview of the results of running a single . It shows the following data:

Average E-value over time: The average E-value of the societies in the simulation at each time step. Note that the E-value is calculated for the chosen part of the society, and then the result is averaged over all the trials. The popup menu in the upper right lets the user look at the average E-values of subsets of the society, picked out in terms of their E-value. Thus the Bottom Quintile contains the 20% of the inquirers with the lowest E-value, the Lower Quintile contains the next 20%, etc.

Average E-value: The average E-value of the societies created, at the end of the simulation.

Average E-value Δ: The average of the change in E-value of the societies created, between the start and the end of each simulation.

Polarisation: The average standard deviation of E-value in the societies created, at the end of the simulation.

Polarisation Δ: The average change in the standard deviation of E-evalue in the societies created, between the start and the end of each simulation.

Average Bandwagoning %: The percent of all updates where an inquirer's degree of belief has been changed in the opposite direction from her own opinion or inquiry, as a result of communication from others.

Average Bandwagoning Effect: The average difference in what the inquirer's resulting degree of belief would have been without external influence, compared to with it, when such bandwagoning has occured.

# simulations run: The number of trials in the batch simulation.

# steps/simulation: The number of steps each simulation is run, all stages included.

Average # messages sent: The average number of messages sent during each trial.

Average # inquiry results : The average number of inquiry results received during each trial.

Degrees: Opens the Degree Distribution dialog, where the graph-theoretic properties of the generated societies can be studied.

Export EValues: Opens the Statistics Export dialog, where the exact results of the batch simulation can be saved in a spreadsheet format.

Export Topologies: Opens the Topologies Export dialog, from which one can export the network topologies generated in the simulation in Pajek format.
