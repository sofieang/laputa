Batch Simulations

A Batch simulation is a collection of simulations, run from a (possibly randomized) initial state for a specific number of steps, while statistics is collected. Since Laputa's model is a stochastic one, it is only batch simulations that can give enough information to draw stable conclusions from.

The trials field is used to specify the number of simulations that the batch simulator is to run and average over. The batch simulator allows each such simulation to consist of up to four stages, which are executed in succession. Small triangular buttons between the stage selection buttons are used to copy all parameters of one stage to another. The number of steps in each stage are set in the Stage tab. Before each stage, the society's properties are modified according to the settings in the other tabs:

  • Network
  • Inquriers
  • Links
  • E-Values
  • Record

Of these, the E-Value and the Record tab are common to all stages, while the others pertain to the specific stage selected on the left.

Some of the parameters in the Batch window (Starting Belief, Inquiry Accuracy, Inquiry Trust, Listen Chance, Threshold and Trust) correspond directly to properties of inquirers and links, and are used to determine starting values of these, unless overridden by template links' or inquirers' simulation parameters. All of these are represented by various forms of distributions or metadistributions.

There are also buttons to save and load settings for the batch simulation windows, and the same settings can be loaded and saved from the Generate Random Society window as well, whose layout is very similar to that of the batch window.

When a batch simulation has finished, the statistics window is opened to display the results.
