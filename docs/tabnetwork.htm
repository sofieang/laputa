The Network tab

Each society is created in 3 stages:

  1. A number of inquirers are created, with properties as determined from the inquirers tab.
  2. A number of links between these inquirers are added, with properties determined by the links tab.
  3. Finally, a number of inquirers are added one by one, with links from or to those inquirers already in the society.

The network tab determines the network structure of a random society, i.e. the number of inquriers, and how they are linked to one another. It is separated into two parts: population and number of links.

Population

This part of the network tab determines the total number of inquirers, as well as the proportion of inquirers added one by one in step 3, as opposed to added all at one in step 1. It contains two controls:

Population: A probability distribution that determines how many inquirers in total that the generated society will have. The check box Create, if ticked, ensures that an entirely new society will be created. If unticked, no new inquirers will be created in this stage.

Growth: When the On check box is ticked, stage 3 is invoked. This means that for some part of the population determined by the population probability distribution, that part will be added one by one rather than in step 1. There are sliders to determine how large a part of the society is to be grown this way, as well as the balance of links to the new inquirers to add as opposed to links from them. Setting the balance slider to 0 will result in the new inquirers, when added, only being listeners, while setting it to 1 results in them having links going only out of them.

Number of links

This part determines which inquirers are linked to which. It also contains two controls:

Number of links: A probability distribution that determines how many links in total that the generated society will have. This can be applied in a total of five ways, as selected from the two popup menus below the distribution view.

Density / in total: A number [0, 1] is picked from the distribution, and multiplied with the maximal possible number of links.
Density / per inquirer: A number [0, 1] is picked for each inquirer in the society, and multiplied with the maximal possible number of links to that inquirer. These are added up.
Density / per inquirer squared: A number [0, 1] is picked for each ordered pair of inquirers, except those on the diagonal. These are added up.
Number / in total: The number of links is as picked in the distribution, but numbers above n(n + 1), where n is the number of inquirers, are clamped to this number.
Number / per inquirer: The number of links is picked for each inquirer and added up, but numbers above n(n + 1), where n is the number of inquirers, are clamped to this number.
The check box Create, if ticked, ensures that an entirely new set of links will be created, and the ones of the previous stage removed. If unticked, no new links will be created in this stage. The check box Limit links to one per pair, if unchecked, will allow the same link to be added more than once. The result of such an addition is determined as a mean of the properties of the new and the old link, with the exception of the listen chance, which is given as P(lcnew) + P(lcold) - P(lcnew)P(lcold).

Weights: When a link is added in stage 2, the target is picked randomly and uniformly from the current population. However, in both step 2 and step 3, the link's source is determined by a weighted random distribution. The Base weight is that assigned to each inquirer by default. Each of the others is then added when appropriate.

Symmetry: Additional weight added to an inquirer if that inquirer already has a link to it from the target.
Transitivity: Additional weight added to an inquirer if that inquirer already has a link to some inquirer that has a link to the target.
Clustering: Additional weight added to an inquirer for each link it already has that goes from it.
